### Estructura de URLs
- [x] URLs limpias y descriptivas
- [x] Uso de anchors para secciones (#hero, #solutions, etc.)
- [x] Sin parámetros innecesarios
- [ ] Considerar crear páginas separadas para servicios específicos

### Contenido
- [ ] Mínimo 300 palabras en página principal
- [ ] Uso natural de keywords (densidad 1-2%)
- [ ] Contenido único y original
- [ ] Actualizar regularmente (blog recomendado)
- [ ] Incluir FAQs (✓ ya implementado)

### Imágenes
- [ ] Todas las imágenes con atributo `alt` descriptivo
- [ ] Nombres de archivo descriptivos (no IMG_1234.jpg)
- [ ] Formato WebP/AVIF para mejor compresión
- [ ] Lazy loading implementado
- [ ] Tamaño optimizado (< 100KB por imagen idealmente)

### Links
- [ ] Links internos a secciones relevantes
- [ ] Links externos a recursos de autoridad (si aplica)
- [ ] Anchor text descriptivo
- [ ] No broken links
- [ ] Links a redes sociales con rel="noopener"

### Mobile & UX
- [x] Diseño responsive
- [ ] Touch targets > 48x48px
- [ ] Texto legible sin zoom (font-size ≥ 16px)
- [ ] Evitar intersticiales intrusivos
- [ ] Velocidad de carga < 3s (First Contentful Paint)

### Core Web Vitals
Objetivos:
- **LCP (Largest Contentful Paint)**: < 2.5s
- **FID (First Input Delay)**: < 100ms
- **CLS (Cumulative Layout Shift)**: < 0.1

### Seguridad
- [x] HTTPS habilitado
- [x] Security headers implementados
- [ ] Política de privacidad
- [ ] Términos y condiciones
- [ ] Cookie consent (si aplica GDPR/CCPA)
